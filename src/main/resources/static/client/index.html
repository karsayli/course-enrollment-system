📋 Complex Business Operation: "Enroll Student in Course"

Description: This complex operation demonstrates a multi-step business process that involves multiple REST API calls. The operation:

  1. Fetches all available students (GET /api/students)
  2. Fetches all available courses (GET /api/courses)
  3. Checks if enrollment already exists (GET /api/enrollments/exists/student/{id}/course/{id})
  4. Checks course capacity by getting full courses (GET /api/courses/full)
  5. Creates the enrollment (POST /api/enrollments)
  6. Verifies the enrollment was created (GET /api/enrollments/{id})

This demonstrates a real-world scenario where a single business action requires multiple API calls to validate business rules and complete the transaction.
